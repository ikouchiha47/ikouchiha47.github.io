Basic Algorithms

  • Binary Search
  • Bisect Left
  • Bisect Right
  • Minimize Maximum, vice versa
  • Prefix Sum

The pre-condition is for the array to be sorted or sorted/rotated. O(log n) time.

The idea is: To take an item randomly and check if the requested item can be fitted in left side or right side of the selected item.

binary_search

search(nums: List, target):
  lo, hi = 0, nums.length

  while lo < hi:
    mid = (lo + hi) >> 1 # prandomly pick an item
    # or 
    # (lo + hi) // 2
    # math.floor(lo + (hi - lo) / 2)
    # (lo >> 1) + (hi >> 1)

    if target == nums[mid]:
      return True

    # reduce the search space 
    # to check fittment in the region
    if target < nums[mid]:
      lo = mid + 1
    else:
      hi = mid

  return False

For bisect left, you are to get the lowest index in the array, where the element would exists if it wasn’t there, or the first occurance of the target item.

Bisect right is the opposite. Examples:

nums = [2,2,3,3,5,5,5]
bisect_left(nums, 5) # => 4
bisect_left(nums, 1) # => 0
bisect_left(nums, 6) # => 7

bisect_right(nums, 5) # => 7
bisect_right(nums, 1) # => 0
bisect_right(nums, 4) # => 4 # idx to insert at
bisect_right(nums, 6) # => 7

So when trying to find an element with bisect_left and bisect_right, look for the validating the index or item at index.

bisect_left

as long as the target_item <= present_item_in_consideration, keep shrinking the search space tp left.

loop:
  mid = (lo + hi) >> 1
  if target <= nums[mid]: hi = mid # shrinking to left
  else: lo = mid + 1

return lo

bisect_right

as long as target_item >= present_item_in_consideration, keep shirking the space to right

loop:
  mid = (lo + hi) >> 1
  if target >= nums[mid]: lo = mid + 1 # shrinking to right
  else: hi = mid

return lo
